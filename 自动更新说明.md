# FlowPilot 自动更新功能说明

## 概述

FlowPilot v0.3.7 优化了更新提示，发现新版本时会提示用户确认下载。

## 使用方式

### 1. 获取 flow.js

下载最新的 `flow.js` 文件：
- GitHub Release: https://github.com/6BNBN/FlowPilot/releases
- 或者: `curl -L https://raw.githubusercontent.com/6BNBN/FlowPilot/main/dist/flow.js -o flow.js`

### 2. 运行

```bash
node flow.js init
node flow.js next
node flow.js status
# ...其他命令
```

### 3. 发现新版本

当有新版时会提示：

```
🔄 发现新版本: v0.3.7 (当前: v0.3.6)
   输入 y 下载更新，或按其他键跳过
💡 提示: 请运行 node flow.js init 重新初始化
```

输入 `y` 下载最新版本，否则跳过。

### 4. 更新后

下载新版本后，**必须**运行 `node flow.js init` 重新初始化，以确保配置和功能是最新的。

### 5. 跳过更新检查

以下命令不触发更新检查：
- `node flow.js version` - 查看版本
- `node flow.js -h` / `--help` - 查看帮助

## 缓存机制

- 更新检查结果缓存 24 小时
- 缓存位置: `.flowpilot/update-cache.json`
- 首次运行或缓存过期时才会请求 GitHub API
- 强制检查: 删除缓存文件后运行任意命令

## 版本号

- 版本号在构建时注入到 `flow.js` 头部
- 格式: `// FLOWPILOT_VERSION: x.x.x`
- 查看版本: `node flow.js version`

## 常见问题

### Q: 更新后需要重新初始化吗？
A: 是的，必须运行 `node flow.js init` 重新初始化以确保配置最新。

### Q: 没有网络怎么办？
A: 自动跳过更新检查，不影响命令执行。

### Q: 如何强制检查更新？
A: 删除 `.flowpilot/update-cache.json` 后运行任意命令即可重新检查。
